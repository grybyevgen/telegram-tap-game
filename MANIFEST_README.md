# Manifest.json для Telegram Mini App "Тапалка"

## Описание параметров

### Основные параметры:
- **short_name**: "Тапалка" - короткое название приложения (до 12 символов)
- **name**: "Telegram Тапалка Игра" - полное название приложения
- **description**: Описание приложения для поисковых систем и магазинов приложений
- **start_url**: "/index.html" - URL для запуска приложения
- **display**: "standalone" - режим отображения (без браузерной оболочки)
- **orientation**: "portrait" - ориентация экрана (вертикальная)
- **theme_color**: "#667eea" - цвет темы для браузера и системных UI
- **background_color**: "#ffffff" - цвет фона при загрузке приложения

### Иконки (icons):

⚠️ **ВАЖНО**: Убедитесь, что файлы `icon-192x192.png` и `icon-512x512.png` существуют в корне проекта!

- **icon-192x192.png**: Иконка для мобильных устройств (192x192 пикселей)
- **icon-512x512.png**: Иконка для больших экранов и установки PWA (512x512 пикселей)

**purpose: "any maskable"** - означает, что иконка может быть использована в различных формах (круглые, квадратные, адаптивные)

### Shortcuts (ярлыки):
Быстрые действия, доступные из меню приложения:
- **Играть** - быстрый запуск игры

## Как заменить placeholder иконки:

1. **Быстрый способ (браузер):**
   - Откройте `generate-icons.html` в браузере
   - Настройте параметры
   - Скачайте иконки
   - Поместите `icon-192x192.png` и `icon-512x512.png` в корень проекта

2. **С помощью Node.js:**
   ```bash
   npm install sharp
   node create-icons.js
   ```

3. **Вручную:**
   - Создайте PNG файлы размером 192x192 и 512x512 пикселей
   - Назовите их `icon-192x192.png` и `icon-512x512.png`
   - Поместите в корень проекта (там же, где manifest.json)

## Проверка manifest.json:

3. Перейдите в Application → Manifest
4. Проверьте, что все иконки загружаются без ошибок

## Troubleshooting:

### Проблема: Иконки не отображаются
**Решение:**
- Убедитесь, что файлы имеют правильные имена (`icon-192x192.png`, не `icon-192.png`)
- Проверьте, что файлы находятся в той же директории, что и `index.html`
- Проверьте путь в `manifest.json` (должен быть относительным: `"icon-192x192.png"`)
- Очистите кеш браузера (Ctrl+Shift+R)

### Проблема: Manifest не загружается
**Решение:**
- Убедитесь, что `<link rel="manifest" href="manifest.json">` есть в `index.html`
- Проверьте, что `manifest.json` является валидным JSON (используйте JSON validator)
- Проверьте MIME-тип файла на сервере (должен быть `application/manifest+json`)

### Проблема: Приложение не устанавливается как PWA
**Решение:**
- Убедитесь, что приложение открывается по HTTPS (или localhost)
- Проверьте, что все иконки доступны
- Убедитесь, что `start_url` указывает на существующий файл
- Проверьте, что `scope` охватывает все нужные страницы
